#Build System
If you detect a Makefile at the root directory of the project, there is no need to create any further build code (execpt as specifically requested by me).  Otherwise, we will use the build system normally used for the main computer language mentioned in the `t:` directive (or defaulted, if that is missing).  however, you should also create a Makefile in the root directory of the program which calls the native system.  in the Makefile, the default `all` target should build the solution in debug mode (if applicable).  there should also be the following PHONY targets: `release`, `clean`, `distclean`, `install-debug`, and `install`.  the `distclean`option should run the normal `clean` function of the native build tool (or perform nothing if there is no build, such as for an interpreted python script) and it should also delete any additional files generated by an initial build, such as `Cargo.lock` in the case of the rust programming language or pre-compiled headers in the case of c++, etc. the two `install` targets should silently ensure there is a `bin` directory in the current user's home directory, then ensure the executable file has file mode 0774 set, then copy the executable file to the $HOME/bin/ directory.
